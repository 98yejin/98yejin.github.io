## Module 1 Summary

- Cloud computing is the delivery of on-demand computing resources over the internet on a pay-as-you-go basis; resources are dynamically assigned and reassigned among multiple users and scale up and down in response to users’ needs.

: 클라우드 컴퓨팅은 종량제 방식으로 인터넷을 통해 주문형 컴퓨팅 리소스를 제공하는 것이다. 리소스는 여러 사용자에게 동적으로 할당 및 재 할당되며 사용자의 요구에 따라 확장 및 축소된다.

- The origins of cloud computing can be traced back to the mainframes of the 1950s, with virtualization technologies and hypervisors serving as catalysts for the emergence of modern-day cloud computing.

: 클라우드 컴퓨팅의 기원은 1950 년대의 메인 프레임으로 거슬러 올라갈 수 있으며 가상화 기술과 하이퍼 바이저는 현대 클라우드 컴퓨팅의 출현을위한 촉매 역할을 한다.

- Organizations must consider their business needs, investment viability, and risk capacity in order to create a cloud adoption strategy that delivers desired benefits without causing business disruptions and security, compliance, or performance issues.

: 조직은 비즈니스 중단 및 보안, 규정 준수 또는 성능 문제를 일으키지 않고 원하는 이점을 제공하는 클라우드 채택 전략을 수립하기 위해 비즈니스 요구 사항, 투자 가능성 및 위험 용량을 고려해야한다.

- Cloud adoption is growing faster than predicted. Driving this technological wave are cloud service providers with a host of services ranging from Infrastructure, Platform, and Software services. Some major Cloud providers of our times include AWS, Alibaba Cloud, Google, IBM Cloud, and Microsoft Azure.

: 클라우드 채택이 예상보다 빠르게 증가하고 있다. 이러한 기술 물결을 주도하는 것은 인프라, 플랫폼 및 소프트웨어 서비스에 이르는 다양한 서비스를 제공하는 클라우드 서비스 제공 업체다. 우리 시대의 주요 클라우드 제공 업체에는 AWS, Alibaba Cloud, Google, IBM Cloud 및 Microsoft Azure가 있다.


#### Quiz를 위한 [NIST](https://www.nist.gov/news-events/news/2011/10/final-version-nist-cloud-computing-definition-published)의 Cloud Computing 정의

- Cloud computing is a relatively new business model in the computing world. According to the official NIST definition, "cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction."

#### Hypervisor의 정의

- A hypervisor is a small software layer that enables multiple operating systems to run alongside each other, sharing the same physical computing resources.

- A hypervisor also separates the Virtual Machines logically, assigning each its own slice of the underlying computing power, memory, and storage, preventing the virtual machines from interfering with each other.

## Module 3 Summary

- Cloud computing allows us to utilize technology as a service, leveraging remote resources on-demand, on a pay-as-you-model. There are three main service models available on the cloud—Infrastructure-as-a-Service, Platform-as-a-Service (PaaS), and Software-as-a-Service (SaaS).

: 클라우드 컴퓨팅을 사용하면 종량제 모델에서 온 디맨드로 원격 리소스를 활용하여 기술을 서비스로 활용할 수 있다. 클라우드에서 사용할 수있는 세 가지 주요 서비스 모델은 Infrastructure-as-a-Service, Platform-as-a-Service (PaaS) 및 Software-as-a-Service (SaaS)이다.

-- IaaS provides the fundamental compute, network, and storage resources for customers on-demand.

: IaaS는 온 디맨드 고객에게 기본적인 컴퓨팅, 네트워크 및 스토리지 리소스를 제공한다.

-- PaaS provides customers the hardware, software, and infrastructure to develop, deploy, manage, and run applications created by them or acquired from a third-party.

: PaaS는 고객이 생성했거나 타사에서 획득 한 애플리케이션을 개발, 배포, 관리 및 실행할 수있는 하드웨어, 소프트웨어 및 인프라를 고객에게 제공한다.

-- SaaS provides access to users to a service provider’s cloud-based software. Users simply access the applications on Cloud while the Cloud provider maintains the infrastructure, platform, data, application code, security, availability, and performance of the application.

: SaaS는 사용자에게 서비스 제공 업체의 클라우드 기반 소프트웨어에 대한 액세스를 제공한다. 사용자는 클라우드에서 애플리케이션에 액세스하기 만하면 클라우드 제공 업체는 애플리케이션의 인프라, 플랫폼, 데이터, 애플리케이션 코드, 보안, 가용성 및 성능을 유지한다.

- Deployment models indicate where the infrastructure resides, who owns and manages it, and how cloud resources and services are made available to users. There are three main deployment models available on the cloud—Public, Private, and Hybrid.

: 배포 모델은 인프라가있는 위치, 인프라를 소유 및 관리하는 사람, 클라우드 리소스 및 서비스가 사용자에게 제공되는 방식을 나타낸다. 클라우드에서 사용할 수있는 세 가지 주요 배포 모델은 public, private 및 hybrid이다.

-- In the Public cloud model, the service provider owns, manages, provisions, and maintains the physical infrastructure such as data centers, servers, networking equipment, and storage, with users accessing virtualized compute, networking, and storage resources as services.

: Public cloud 모델에서 서비스 제공 업체는 데이터 센터, 서버, 네트워킹 장비 및 스토리지와 같은 물리적 인프라를 소유, 관리, 프로비저닝 및 유지하며 사용자는 가상화 된 컴퓨팅, 네트워킹 및 스토리지 리소스에 서비스로 액세스한다.

-- In the Private cloud model, the provider provisions the cloud infrastructure for exclusive use by a single organization. The private cloud infrastructure can be internal to the organization and run or on-premises. Or, it can be on a public cloud, as in the case of Virtual Private Clouds (VPC) and be owned, managed, and operated by the cloud provider.

: Private cloud 모델에서 제공자는 단일 조직에서만 사용할 수 있도록 클라우드 인프라를 프로비저닝한다. Private cloud 인프라는 조직 내부에있을 수 있으며 실행 또는 on-premises 일 수 있다. 또는 Virtual Private Cloud (VPC)의 경우처럼 Public cloud에있을 수 있으며 클라우드 제공 업체가 소유, 관리 및 운영 할 수 있다.

-- In the Hybrid cloud model, an organization’s on-premise private cloud and third-party, public cloud is connected as a single, flexible infrastructure leveraging the features and benefits of both Public and Private clouds.

: (조직의 on-premises private cloud 및 third-party)Hybrid cloud 모델에서 Public cloud는 Public 및 Private cloud의 기능과 이점을 모두 활용하는 유연한 단일 인프라로 연결된다.
